# UI Audit & Engine Enhancements - 2026-07-18

## Overview
Today's session focused on resolving critical performance bottlenecks within the core audio engine and UI rendering, ensuring Nimbus can operate efficiently on older hardware while retaining professional DAW continuity.

## Modifications
1. **Audio Engine Playback & Mute Logic**:
   - Addressed an issue where muting a track paused the playhead and stopped plugin processing.
   - Refactored `Track::processBlock()` to remove the early return upon mute. The track now fully computes its audio (allowing plugin tails and playhead sync to continue) but dynamically silences the summation into the master bus.
2. **Detail View Optimization (Clip View)**:
   - Addressed a severe CPU freeze when expanding the AudioClipViewComponent for large tracks.
   - Refactored `AudioClipContent::paint()` to map the visible `juce::Graphics` clip bounds to exact waveform times. 
   - `AudioThumbnail` now exclusively draws the subset of the waveform currently visible on-screen, preventing memory over-allocation and saving thousands of rendering cycles.
   - Applied identical viewport culling to the Warp Markers rendering loop.

## Issues Encountered
- **Clip Properties & Detail View**: The Ableton-style clip properties panel controls were not functionally responsive, and changes to the global tempo caused unintended behaviors to clips. (Resolution: Temporarily hid the clip detail view child panels to prevent disruption while we rethink the architecture).
- **Pitch Correction DSP Glitches**: The initial granular/time-domain delay line approach caused "flashing lights" behavior where the audio was severely artifacted and drifted incorrectly during unpitched sections. (Resolution: After attempting Phase Vocoder and ACF Time-Domain solutions, the plugin was abandoned and completely removed from the project due to unresolved tracking and shifting issues on complex/polyphonic vocal samples).
- **Live Mode UI Sizing**: The Right Panel dials in the Pitch Correction plugin were previously overlapping and clipping their text. (Resolution: Widened the Live Mode section and expanded the dial hitboxes to correctly display text and graphics. Moved the Reset button to the title bar).

## Status
- Verified playback engine latency and state stability.
- Verified waveform viewport efficiency under zoom and scroll.
- Temporarily disabled Clip Detail View properties until architecture is revised.
- Moving forward to implement the Pitch Correction component.
