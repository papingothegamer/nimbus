# Daily Audit: July 23, 2026

## Focus: Track Grouping — Duplicated Clip Not Rendering in Timeline

### Problem Statement
When a track is duplicated within a track group (or when two tracks are grouped after duplication), the second track's clip waveform does not appear in the timeline grid. The audio plays correctly from both tracks, the mixer shows two channels with VU activity, and the track header is visible — but the `ClipComponent` in the second track's lane is invisible.

A red-border debug test on July 20 confirmed that the `ClipComponent` either does not exist, or has zero/invalid bounds where the second clip should render.

### Actual Root Cause & Solution (Resolved)

The missing clip bug was caused by an insidious and obscure C++ idiosyncrasy involving `std::vector::insert` and `std::initializer_list`.

When `TimelineProject::groupTracks` (and `insertTrack`) attempted to create an empty list of clips for the incoming track, the code used an empty pair of braces:
```cpp
trackClips.insert(trackClips.begin() + firstIndex, {});
```

Because `trackClips` is a vector of vectors (`std::vector<std::vector<AnyClipPtr>>`), the intention was for `{}` to default-construct a single empty `std::vector<AnyClipPtr>` and insert it. However, the compiler resolved `{}` to the `std::initializer_list` overload of `std::vector::insert`. Because the initializer list was completely empty, the `insert` call essentially did nothing—it inserted **zero elements**.

As a result:
1. The `tracks` array grew by 1 element (for the Group Track).
2. The `trackClips` array did *not* grow at all.
3. This created an immediate index desync: the Group Track accidentally inherited the Original Track's clips, the Original Track inherited the Duplicate's clips, and the Duplicate Track was left pointing to an out-of-bounds index (rendering 0 clips).
4. The audio engine continued to play the audio properly because it had already cached the old valid state in its `TrackSourceNode` objects before the desync occurred.

#### Fix Implemented
Replaced the empty `{}` with explicit default constructors: `std::vector<AnyClipPtr>()`.
```cpp
trackClips.insert(trackClips.begin() + index, std::vector<AnyClipPtr>());
```
This forces the vector to properly insert one empty element, keeping `tracks` and `trackClips` perfectly synchronized at all times.

### Compressor Plugin UI Remodel
- **Objective**: Remodel the stock Compressor plugin to match professional DAW aesthetics, using Audacity's layout and presets as a reference point.
- **Data Model & DSP Updates**: Added a **Make-up Gain** parameter. Integrated live `currentInputLevel` and `currentGainReduction` tracking directly into the `processBlock` thread.
- **Live Transfer Curve Graph**: Created the `CompressionGraphComponent` that visually plots the compressor's transfer curve dynamically based on Threshold, Ratio, and Make-up gain. Implemented a dancing "live dot" that visualizes real-time gain reduction.
- **Factory Presets**: Migrated Audacity's core compressor presets (General, Mastering, Vocal, Instrument categories). Implemented the `PresetCarousel` layout pattern (from `ReverbPlugin`) in the top bar to replace the old preset ComboBox.
- **Dynamic Layout Refactor**: Updated the internal flexbox/bounds layout inside `CompressorPluginEditor` to scale mathematically, ensuring the 5 parameter dials fit perfectly within the constrained heights of the `DeviceChainComponent` without getting clipped.

### Computer MIDI Keyboard & Track Routing
- **QWERTY MIDI Implementation**: Implemented a standalone `ComputerMidiController` module mapping standard QWERTY keys to a two-octave piano layout (starting at 'A' for C). Handles velocity and octave shifting, as well as time-based smoothed Modulation Wheel and Pitch Bend inputs using the arrow keys via a 60Hz UI timer.
- **Global KeyListener**: Hooked the `ComputerMidiController` into the main application via `NimbusEngine` and `MainWindow` as a global `juce::KeyListener`. It can be toggled via the piano icon in the `TopToolbarComponent`.
- **MIDI Track Processing Restrictions**: Edited the `Track::processBlock` core loop to enforce DAW-standard MIDI routing constraints. MIDI tracks will now only receive live MIDI events and show activity on the UI meters if the track is armed **and** an instrument VST is loaded onto the track.
- **MIDI Settings Menu Remodel**: Revamped the MIDI Setup tab in `SettingsMenuComponent` by replacing the generic `juce::AudioDeviceSelectorComponent` with a custom `CustomMidiSettingsComponent` that dynamically renders stylish toggles for each available hardware MIDI instrument plugged into the user's computer.
